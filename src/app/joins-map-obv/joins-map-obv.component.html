forkJoin

Runs multiple observables in parallel

Emits only once when all observables complete successfully

If any observable fails → error block is called

switchMap

Cancels previous API call when a new one is triggered

Keeps only the latest request result

Best for live search boxes


mergeMap

Runs inner observables in parallel

All results are returned (no cancellation)

Good for nested/parallel API calls

Takes all successful (positive) calls and ignores failed calls

concatMap

Executes observables one after another (sequentially)

Ensures order is maintained (waits for one to finish before next starts)

exhaustMap

Ignores new emissions while the current observable is still running

Prevents duplicate calls (e.g., multiple login clicks)

Waits until current request completes before accepting new ones


Summary

forkJoin → all together, emit once

switchMap → latest only (cancels old)

mergeMap → parallel, returns all successful calls, ignores failed ones

concatMap → sequential, ordered

exhaustMap → ignores new until current completes
