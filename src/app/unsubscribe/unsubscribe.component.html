Unsubscribing Basics

It is important to prevent memory leaks. Because everything we subscribe gets stored in our local memory.

Way 1: Single Subscription

When there is only one subscription, store it in a single Subscription object and call unsubscribe() inside ngOnDestroy().

Way 2: Multiple Subscriptions

For multiple subscriptions, push them into an array of Subscription[]. On destroy, iterate over the array and unsubscribe all.

Way 3: Using takeUntil

Create a Subject (e.g. subTakeUntil) and use it with takeUntil() in the pipe. When the component is destroyed, call next() on the subject to complete all active subscriptions.

Way 4: Using take

Use the take(n) operator to automatically complete the subscription after receiving n values. This prevents memory leaks without manual unsubscription.

Way 5: Using Async Pipe

Bind the observable directly to the template with the async pipe. Angular automatically subscribes and unsubscribes when the component is destroyed.
