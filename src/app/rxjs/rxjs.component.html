Rxjs Basics

RXJS stands for Reactive Extension for Javascript.

RXJS is an external library for reactive programming.

It is a way to handle asynchronous operations, events and data streams.

It is heavily used in Angular's core features (like HttpClient, FormControl.valueChanges, and routing) all use Observables under the hood.

Why RxJS in Angular?

Working with HTTP:Working with HTTP: All HttpClient methods return Observables

Form input tracking: valueChanges returns an Observable

Route parameters: ActivatedRoute.paramMap is an Observable

State management: RxJS is the foundation of libraries like NgRx

Building responsive UIs: You can react to user inputs, data changes, and server responses easily using RxJS.

Observables

Observable does not emit by itself; you have to use next() to emit.

Observables can emit multiple values over time.

They can complete, error out, or keep streaming.

We need to subscribe to start receiving values.
